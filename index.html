Geometric Calculation & Parametrization for Dome Reconstruction

Collaborators: Arch. Marcos Ontiveros, PhD Juan Gerardo Oliva Salinas

Context: Post-earthquake reconstruction · Heritage conservation

Following the 2017 earthquake that severely damaged the main church of Axochiapan, this project involved advanced geometric calculation and parametric modeling using Grasshopper to facilitate the accurate reconstruction of the historic dome.

The seismic event caused significant structural damage to the 18th-century church's dome, requiring precise digital documentation and parametric modeling to guide restoration efforts. Using photogrammetry and laser scanning technologies, we captured the existing conditions and developed a parametric model that respected the original architectural intent while incorporating necessary seismic reinforcements.

Technical Approach

  • Digital Documentation: Laser scanning and photogrammetry of damaged structure
  • Parametric Modeling: Grasshopper definition for dome geometry reconstruction
  • Structural Analysis: Integration with finite element analysis plugins
  • Material Optimization: Parametric optimization of brick patterns and placement
  • Construction Guidance: Generation of fabrication templates and assembly sequences

Research Context

The 2017 Puebla earthquake (7.1 magnitude) damaged numerous historical buildings throughout central Mexico, with churches being particularly vulnerable due to their age and construction techniques. The reconstruction of Axochiapan's main church represents an important case study in combining traditional restoration methods with contemporary computational design approaches.

Project Significance

This project demonstrates how computational design and parametric modeling can serve heritage conservation, particularly in post-disaster scenarios where traditional documentation methods may be insufficient. The Grasshopper-based workflow allowed for accurate reproduction of complex geometric relationships and integration of seismic reinforcement strategies within historical constraints.

Baggage Carousel Calculator

Developer: Eduardo Luce · 2025

Category: Grasshopper Plugin · Parametric Airport Design

📖 Description

The Baggage Handling System (BHS) Carousel Calculator is a design tool for quickly estimating baggage reclaim carousel layouts based on aircraft mix and IATA standards.

It helps planners and designers test different scenarios for narrow-body and wide-body aircraft, calculate the number and dimensions of required carousels, visualize their geometry in Grasshopper, and estimate total passenger capacity and floor area requirements.

⚠️ Disclaimer: Developed by Eduardo Luce (2025) as a Work-in-Progress (WIP) research and planning tool. This tool is not intended for operational deployment without professional verification of all data.
📦 Download Plugin (ZIP, 2.3 MB)

Includes .ghuser file + PDF documentation

🎛️ Input Parameters

Narrow Count (int) Number of narrow-body aircraft (e.g., A320/B737 class)
Wide Count (int) Number of wide-body aircraft (e.g., B777, A350 class)
Standard Length (float) Target equalized carousel length (m) for standard sizing
Draw Mode (int) 1 = Standard only | 2 = Variable only | 3 = Both

📤 Outputs

Carousel Outlines Outer + inner offset curves grouped in pairs
Clearance Zones Clearance boundary curves around each carousel
Estimated Floor Area Sum of carousel footprint + clearances (m²)
Passenger Capacity Estimated passengers based on aircraft input

📚 References & Standards

IATA Airport Development Reference Manual, 9th Edition (2014)
FAA Advisory Circular 150/5360-13A, Planning and Design Guidelines
ICAO Annex 14 - Aerodromes, Volume I - Aerodrome Design and Operations

⚙️ Installation Guide

  1. Save the provided .gha or Grasshopper user object (.ghuser)
  2. Place it in your Grasshopper Libraries folder
  3. Restart Grasshopper
  4. The BHS Carousel Calculator will appear as a custom component
